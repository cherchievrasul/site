# PLAN-01-02: Деплой на GitHub Pages

**Wave:** 2
**Depends on:** 01-01
**Autonomous:** no (требует CNAME-запись в DNS от пользователя)
**Requirements:** DEPLOY-01, BUILD-01

## Objective

Настроить GitHub Actions для авто-деплоя в GitHub Pages при пуше, привязать домен.

## must_haves

- `npm run build` завершается без ошибок
- GitHub Actions деплоит сайт в Pages при `git push`
- Сайт открывается по `https://cherchievrasul.github.io/site/`
- CNAME-файл с доменом пользователя присутствует

---

## Task 1: GitHub Actions — деплой в Pages

<read_first>
- file: package.json (скрипт build)
</read_first>

<acceptance_criteria>
- `.github/workflows/deploy.yml` существует и содержит workflow с триггером `push` на `master`
- Workflow выполняет `npm ci`, `npm run build`, деплоит `dist/` через `actions/upload-pages-artifact` и `actions/deploy-pages`
- `vite.config.js` содержит `base: '/site/'` (имя репозитория для GitHub Pages)
</acceptance_criteria>

<action>
1. Создать `.github/workflows/deploy.yml`:
   - name: Deploy to GitHub Pages
   - on: push branches [master]
   - jobs: build → checkout → setup-node → npm ci → npm run build → upload artifact → deploy
   - Использовать: `actions/configure-pages@v4`, `actions/upload-pages-artifact@v3`, `actions/deploy-pages@v4`
   - artifact path: `dist/`

2. Обновить `vite.config.js`:
   - Добавить `base: '/site/'` (путь репозитория на GitHub Pages)

3. Создать `public/CNAME` с доменом пользователя (плейсхолдер: `example.com` — заменить на реальный домен перед пушем)
</action>

---

## Task 2: [ЧЕКПОИНТ] Пуш и привязка домена

<read_first>
- file: .github/workflows/deploy.yml (workflow деплоя)
- file: public/CNAME (файл домена)
</read_first>

<acceptance_criteria>
- `git remote get-url origin` показывает `https://github.com/cherchievrasul/site.git`
- `git push origin master` проходит успешно
- GitHub Actions запускается и завершается зелёным
- Сайт открывается по `https://cherchievrasul.github.io/site/`
</acceptance_criteria>

<action>
**ЧЕКПОИНТ: перед пушем запросить у пользователя:**
- Точный домен (который куплен на beget)

После получения домена:
1. Записать домен в `public/CNAME`
2. `git add . && git commit -m "deploy: github pages + CNAME"`
3. `git push origin master`
4. Проверить GitHub Actions — дождаться зелёного билда
5. Открыть `https://cherchievrasul.github.io/site/` — убедиться что заглушка работает

**Пользователю после деплоя:**
В DNS-панели beget добавить CNAME-запись:
- Имя: `www` (или `@`)
- Значение: `cherchievrasul.github.io`
- TTL: 3600 (или по умолчанию)

DNS обновляется от 5 минут до 24 часов.
</action>
