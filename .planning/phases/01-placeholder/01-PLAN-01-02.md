# PLAN-01-02: Деплой на Beget

**Wave:** 2
**Depends on:** 01-01
**Autonomous:** no (требует FTP-доступы от пользователя)
**Requirements:** DEPLOY-01, BUILD-01

## Objective

Собрать проект, загрузить через FTP на Beget в `public_html/`, убедиться что сайт открывается по домену.

## must_haves

- `npm run build` завершается без ошибок
- Файлы из `dist/` загружены в `public_html/` на Beget
- При переходе по домену открывается страница-заглушка (не ошибка, не дефолтная страница хостинга)
- Русский текст без кракозябр
- HTTPS работает

## Task 1: Подготовка .htaccess и FTP-окружения

<read_first>
- file: .planning/research/PITFALLS.md (Pitfall 1: FTP directory, Pitfall 2: encoding)
</read_first>

<acceptance_criteria>
- `public/.htaccess` содержит: `AddDefaultCharset UTF-8`, `Options -Indexes`, HTTPS-редирект
- `scripts/deploy.mjs` существует и использует `basic-ftp` для загрузки `dist/`
- `.env.example` документирует необходимые переменные: `FTP_HOST`, `FTP_USER`, `FTP_PASSWORD`, `FTP_REMOTE_ROOT`
- `.gitignore` содержит `.env` и `dist/`
</acceptance_criteria>

<action>
1. Создать `public/.htaccess`:
```
AddDefaultCharset UTF-8
Options -Indexes

RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}/$1 [R=301,L]
```
2. Установить `npm install basic-ftp` (для FTP-скрипта)
3. Создать `scripts/deploy.mjs` — скрипт, который:
   - Читает `.env` (через `process.env`)
   - Подключается к FTP через `basic-ftp`
   - Загружает все файлы из `dist/` в `FTP_REMOTE_ROOT` (по умолчанию `public_html/`)
   - Выводит прогресс и результат
4. Создать `.env.example`:
```
FTP_HOST=your-server.beget.com
FTP_USER=your-ftp-login
FTP_PASSWORD=your-ftp-password
FTP_REMOTE_ROOT=public_html/
```
5. В `.gitignore` добавить: `.env`, `dist/`
</action>

## Task 2: [ЧЕКПОИНТ] FTP-деплой и проверка

<read_first>
- file: scripts/deploy.mjs (скрипт деплоя)
- file: .env (FTP-доступы — запросить у пользователя)
</read_first>

<acceptance_criteria>
- `npm run build && node scripts/deploy.mjs` завершается без ошибок
- При переходе по домену открывается страница-заглушка
- `curl -I https://domain.ru` показывает `content-type: text/html; charset=UTF-8`
- `curl -I https://domain.ru` показывает HTTP 200 (не 403, не 404)
- На реальном телефоне страница открывается и выглядит корректно
</acceptance_criteria>

<action>
**ЧЕКПОИНТ: перед выполнением запросить у пользователя:**
- FTP-хост (из панели Beget → FTP-доступ)
- FTP-логин
- FTP-пароль
- Путь к корню сайта (обычно `public_html/`)
- Домен для проверки

После получения доступов:
1. Создать `.env` с реальными значениями
2. Выполнить `npm run build`
3. Выполнить `node scripts/deploy.mjs`
4. Проверить через браузер: открыть домен, убедиться что страница работает
5. Проверить через `curl -I https://domain.ru` кодировку и HTTPS
6. Открыть на мобильном телефоне — убедиться в адаптивности
</action>
